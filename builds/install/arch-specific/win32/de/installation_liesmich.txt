Firebird Datenbank Server 2.0.5
==================================================


  ** WICHTIG **

  Die ODS hat sich gegenüber Firebird 1.5 geändert.
  Bitte beachten Sie den Hinweis unten bzgl. der neuen
  ODS und der Installation über eine ältere Version von
  Firebird.

  ** ******* **

Dieses Dokument ist eine Hilfestellung um dieses Paket
von Firebird 2.0.5 auf einer Win32 Plattform zu installieren.
Diese Hinweise beziehen sich ausschließlich auf das
Installationspaket, und nicht auf Firebird 2.0.5 allgemein.

Vor der Installation
--------------------

Es wird empfohlen, dass Sie alle älteren Versionen von
Firebird oder InterBase DEINSTALLIEREN, bevor Sie dieses
Paket installieren.


Neue ODS 11.0
-------------

Firebird 2.0 verwendet eine neue ODS 11.0, die komplett
inkompatibel mit der ODS 11.0 von früheren Alpha Releases
ist.

Falls Sie dieses Release von Firebird 2.0 zum erstenmal
über ein Firebird 2 Alpha Release installieren, so empfehlen
wir Ihnen, dass Sie dazu ein neues Verzeichnis verwenden.
Der Grund dafür ist, dass eine Deinstallation die existierende
security2 Datenbank nicht entfernt. Diese ältere Version
verwendet jedoch eine inkompatible ODS und somit ist auch
keine Verbindung zum Server möglich.


Installation auf Systemen mit Firebird 1.5
------------------------------------------

Die Sicherheitsdatenbank wurde geändert. Die neue
Datenbank heißt nun security2.fdb und verwendet nun einen
stärkeren Algorithmus zur Passwortverschlüsselung. Nähere
Informationen zu den neuen Security-Features finden Sie in
den Release Notes.

Die neue Datenbankstruktur und die veränderte Verschlüsselung
haben jedoch einige Migrationsprobleme zur Folge, falls
Installationen eine Vielzahl an Firebrd-Benutzer verwenden.
Die Installationsroutine versucht nicht, diese Migration
durchzuführen. Dies muss manuell durchgeführt werden. Eine
mögliche Migrationstechnik ist in misc/upgrade/security/
angeführt.


Installation auf Systemen mit Firebird 2.0
------------------------------------------

Zusätzlich zur ODS-Änderung wird die Setup-Routine einen
laufenden Firebird 2.0 Alpha oder Beta Server als Dienst
nicht automatisch erkennen. Daher bekommen Sie keine Warnung,
bis der Installer versucht aktive Dateien zu überschreiben.


Neue Features des Installers
----------------------------

o Falls eine existierende firebird.conf vorgefunden wird,
  wird die Default-Konfigurationsdatei unter folgendem
  Dateinamen installiert:

  firebird.conf.default.

  Die existierende firebird.conf Datei bleibt davon unberührt.
  Dies stellt sicher, dass Zugriffsrestriktionen auch nach einem
  Update sichergestellt sind.


o Mehrsprachigkeit wurde dem Installer hinzugefügt. Dies betrifft
  allerdings NUR den Installationsprozess.

  HINWEIS: Die verfügbaren Sprachen werden anhand der verwendeten
  lokalisierten Windows Version ermittelt. Dies bedeutet zum
  Beispiel, dass für Benutzer in Westeuropa gewöhnlich die
  slawischen Sprachen nicht verfügbar sind.

Keine zum jetzigen Zeitpunkt.


Deinstallation
--------------

o Dieses Paket sollte mit der mitgelieferten Deinstallationsroutine
  entfernt werden. Dies kann über die Systemsteuerung durchgeführt
  werden. Alternativ dazu kann dies auch direkt durch das Ausführen
  von unins000.exe im Installationsverzeichnis erfolgen.

o Falls Firebird als Anwendung läuft (anstatt eines Dienstes), so
  wird empfohlen, dass der Server manuell gestoppt wird, bevor die
  Deinstallationsroutine ausgeführt wird. Der Grund dafür ist, dass
  der Deinstaller eine laufende Anwendung nicht beenden kann. Falls
  ein Server während der Deinstallation läuft, wird diese mit Fehlern
  abgeschlossen. Die übrig gebliebenen Dateien müssen im Anschluss
  daran manuell entfernt werden.

o Die Deinstallation entfernt folgende Dateien NICHT:

  - aliases.conf
  - firebird.conf
  - firebird.log
  - security2.fdb

  Dies ist beabsichtigt. Diese Dateien sind höchstwahrscheinlich
  von Benutzern geändert worden und werden vermutlich zu einem
  späteren Zeitpunkt benötigt, falls Firebird neu installiert
  wird. Diese Dateien können gelöscht werden, wenn Sie dafür keine
  Verwendung mehr haben.


Weitere Hinweise
----------------

  Firebird benötigt WinSock2. Alle Win32 Plattformen, ausgenommen
  Win95, sollten WinSock2 installiert haben. Ein Test, ob sich die
  Winsock2 Bibliothek auf dem Rechner befindet, wird während der
  Installation durchgeführt. Falls diese Bibliothek nicht vorgefunden
  wird, so wird die Installation fehlschlagen. Sie können diesen Link
  besuchen:

	http://support.microsoft.com/default.aspx?scid=kb;EN-US;q177719

  um herauszufinden, wie ein Upgrade erfolgen kann.
